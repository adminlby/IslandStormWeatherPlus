你是一名资深 Minecraft Paper 插件开发者，请帮我完整开发一个 Minecraft 高级天气系统插件。

项目名称：IslandStorm
服务端版本：Paper 1.20.6
Java 版本：Java 21
构建工具：Gradle Kotlin DSL
插件类型：服务端插件，兼容 Paper / Purpur / Leaf 1.20.6

插件目标：
为一个 100 小时荒岛生存直播服务器开发高级天气系统。插件不仅要控制 Minecraft 原版天气，还要加入风速、风向、天气预报、区域天气、玩家姿态影响、鞘翅飞行影响、强风吹动玩家、台风路径、极端风暴路径、方块破坏、精美 HTML 天气预览卡、网页控制台等功能。

1. CLEAR：晴天
2. CLOUDY：多云
3. RAIN：小雨
4. HEAVY_RAIN：暴雨
5. THUNDERSTORM：雷暴
6. FOG：浓雾
7. WINDY：大风
8. TYPHOON：台风
9. EXTREME_STORM：极端风暴

每种天气都要包含：

* 描述
* 图标
* 默认持续时间
* 默认风速
* 默认风向
* 是否下雨

* 危险等级
* 是否允许破坏方块
* 默认破坏强度
* 默认影响半径

三、风系统

插件必须实现独立风系统。

风速单位使用 km/h。

风向支持：

* N
* NE
* E
* SE
* S
* SW
* W
* NW

风速等级：

0-10 km/h：无影响
11-30 km/h：轻微影响
31-60 km/h：明显影响
61-90 km/h：强风
91-120 km/h：危险强风
120+ km/h：极端危险

风的实际用途：

1. 玩家在地面行走时，强风会轻微改变移动姿态和速度。
2. 风速较大时，玩家会被风向推动。
3. 风太大时，玩家会被吹走。
4. 玩家使用鞘翅飞行时，风会影响飞行姿态和飞行方向。
5. 顺风时，鞘翅速度略微增加。
6. 逆风时，鞘翅速度下降。
7. 侧风时，鞘翅飞行方向会被轻微偏移。
8. 极端风暴中，玩家可能会失控偏航。
9. 所有效果必须可在 config.yml 中开关和调节强度。

请监听 PlayerMoveEvent 或使用定时任务，并使用 Bukkit Vector 来实现风对玩家速度的影响。

注意性能优化：

* 不要每 tick 对所有玩家做重计算。
* 使用定时任务，例如每 5 tick 或 10 tick 更新一次。
* 只处理在线玩家。
* 只处理生存/冒险模式玩家。
* 管理员权限玩家可以绕过风影响。

四、区域天气系统

插件必须支持区域天气。

区域天气要求：

1. 每个区域可以设置独立天气。

3. 每个区域可以设置独立风向。
4. 每个区域可以设置危险等级。

8. 区域可以通过游戏内指令创建。
9. 区域也可以通过网页控制台地图画框创建。

七、网页控制台鉴权机制

必须新增独立配置文件：

web-users.yml

用于存放网页控制台用户。

要求：

1. 支持多个用户。
2. 每个用户有独立密码 hash。
3. 每个用户有独立权限列表。
4. 不允许明文密码。
5. 默认生成一个 admin 用户。
6. 默认密码不要直接写死为明文。
7. README 中说明如何用指令生成密码 hash。
8. 支持通过游戏后台命令创建用户、修改密码、删除用户。
9. 使用 BCrypt 或 PBKDF2 进行密码 hash。
10. 如果不想引入外部依赖，使用 Java 原生 PBKDF2WithHmacSHA256。
11. 登录成功后生成随机 token。
12. token 有过期时间。
13. token 过期时间可配置。
14. API 请求需要携带 token。
15. token 存在内存中即可，重启后失效。

web-users.yml 示例：

users:
admin:
password-hash: "PBKDF2$sha256$100000$salt$hash"
permissions:
- "*"

password-hash: "PBKDF2$sha256$100000$salt$hash"
permissions:
- "weather.view"
- "weather.set"
- "region.view"
- "region.set"
- "html.generate"
viewer:
password-hash: "PBKDF2$sha256$100000$salt$hash"
permissions:
- "weather.view"
- "region.view"

九、台风与极端风暴路径系统

TYPHOON 和 EXTREME_STORM 必须支持横移路径。

路径要求：

1. 路径由多个点组成。
2. 点使用世界坐标 X/Z。
3. 每个点可以设置预计到达时间。

5. 风暴有影响半径。
6. 风暴半径内玩家受到风影响。
7. 风暴中心附近风力更强。
8. 风暴路径可以通过指令设置。
9. 风暴路径可以通过网页控制台地图拖拽设置。
10. 网页地图上要显示风暴路径线和当前风暴中心位置。

十、方块破坏系统

台风和极端风暴可以破坏方块。

要求：

1. 只有 TYPHOON 和 EXTREME_STORM 默认允许破坏方块。
2. 区域天气可以独立开关方块破坏。
3. 风暴路径可以独立开关方块破坏。
4. 破坏程度可配置。
5. 破坏方块需要性能安全。
6. 不允许一次性扫描大量方块。
7. 使用分批任务。
8. 每次只随机检查少量方块。
9. 可以配置每次检查方块数量。
10. 可以配置破坏概率。
11. 可以配置最大 Y 范围。
12. 可以配置保护方块黑名单。
13. 可以配置只破坏指定白名单方块。
14. 默认不要破坏箱子、床、命令方块、屏障、结构方块、刷怪笼、末影箱、潜影盒。
15. 默认不要破坏 WorldGuard 区域内方块，如果检测到 WorldGuard 存在，需要预留 Hook；第一版可以做占位 Hook，不强制依赖 WorldGuard。
16. 破坏前触发自定义事件，允许其他插件取消。
17. 破坏效果可以选择：

    * 直接变空气
    * 掉落物品
    * 变成掉落方块 FallingBlock
18. 所有效果可在 config.yml 调节。

破坏等级建议：

level 0：不破坏
level 1：只破坏树叶、草、花、火把
level 2：破坏玻璃、木板、栅栏、作物
level 3：破坏部分泥土、沙子、羊毛、轻质方块
level 4：破坏更多自然方块
level 5：高破坏，不建议默认开启

二十一、时间系统：同时支持现实时间和 Minecraft 时间

插件中所有涉及时间的地方，都必须同时支持：

1. 现实世界时间
2. Minecraft 游戏时间

所有时间设置都必须可以选择使用：

* REAL_TIME
* MC_TIME

时间单位必须支持：

现实时间：

* s
* m
* h
* d

Minecraft 时间：

* tick
* ticks
* mcminute
* mchour
* mcday

时间表达示例：

/storm setweather HEAVY_RAIN 30m REAL_TIME
表示现实 30 分钟。

/storm setweather HEAVY_RAIN 6h MC_TIME
表示 Minecraft 时间 6 小时。

/storm setweather CLEAR 1mcday MC_TIME
表示 Minecraft 一天。

/storm path addpoint typhoon-001 -500 200 10m REAL_TIME
表示现实 10 分钟后到达该点。

/storm path addpoint typhoon-001 -500 200 6h MC_TIME
表示 Minecraft 6 小时后到达该点。

如果用户不写时间类型，默认使用 config.yml 中的默认时间模式。

config.yml 需要新增：

time:

show-both-time-in-html: true
show-both-time-in-chat: true

所有涉及持续时间、开始时间、到达时间、预警时间、预报时间的功能，都必须支持 REAL_TIME 和 MC_TIME：

1. 天气持续时间

必须新增 TimeMode 枚举：

REAL_TIME

必须新增 GameTimeUtil 工具类，用于：
